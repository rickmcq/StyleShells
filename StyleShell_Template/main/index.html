The Problem

How does a website creative team deliver all the creative artifacts (files, etc.) for a custom-designed website that is not based on WordPress (or Wix, or SquareSpace, etc.)?

Proposed Solution

Use this Style Shell! The Style Shell website is an 'empty' website that contains all of a website's branding and styles but no content or function (custom code).

The project is used to compose and deliver the site's creative elements to the site owner so that they can take over these elements and integrate them with the content and function to form the complete site.

Alternative: Use Git/GitHub

TO BE EXPANDED: The creative team could use Git/GitHub to contribute derectly to the ongoing project.

How to Use this Project:

  1. (Site Owner) Give a copy of the StyleShell_Template website to your creative team with instructions on how to fill it out with branding and style information.
  2. (Creative Team) Add all branding and creative work to the template site and deliver it back to the owner as StyleShell_CompanyX.
  3. (Site Owner) Integrate content and function with the style shell provided.
NOTE that iterations (updates, versions) can be performed by re-delivering this project. The recipient (site owner) can do a 'diff' to see what's changed and update the 'integrated' site accordingly.

Project contents:

Folders and files used in production

First, let's get these two BootStrap folders out of the way. BootStrap uses these; do not modify them:
  • Folder: Content. A BootStrap folder. Do not modify.
  • Folder: Scripts. A BootStrap folder. Do not modify.
Next, here are folders that the creative team will contribute their content:
  • Folder: images. A standard folder to put all images.
  • Folder: includes. A standard folder to put all css, fonts, and js.
    • css. Contains all cascading style sheets.
    • fonts. Contains any font files needed for the style(s) used.
    • js. Contains any javascript needed for the style(s) used.
    NOTE that these folders may contain other files NOT provided by the creative team.
  • Folder: main. A folder that can be used to either hold the main content, or be an entry point to other content.
  • Folder: media. Used? A standard folder to hold media.

Folders and files NOT used in production

The creative team will also contribute content here:
  • Folder: yDesign. Demos, displays, and palettes of the design elements. See ReadMe.txt ReadMe.md ReadMe.htm.
  • Folder: yInstructions. How to use this template site. See ReadMe.
  • Folder: yTemplates. Starter pages. See ReadMe.
